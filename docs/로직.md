# 추천 번호 30세트가 만들어지는 흐름

> 이 문서는 **개발 내부 로직**이 아니라, 사이트를 사용하는 입장에서  
> **"버튼 한 번 누르면 왜 그 30세트가 나오는가?"** 를 설명합니다.

---

## 1. 전체 흐름 한눈에 보기

```text
   ┌──────────────────────────────────────────────────────────────┐
   │  ① 회차 선택                                                  │
   │     · 사용자가 "몇 회차 기준으로 뽑을까?" 만 고르면 끝          │
   └────────────────────────────┬─────────────────────────────────┘
                                ▼
   ┌──────────────────────────────────────────────────────────────┐
   │  ② 과거 당첨 이력 불러오기                                    │
   │     · 선택한 회차 "이전"의 모든 당첨번호를 서버에서 가져옴      │
   └────────────────────────────┬─────────────────────────────────┘
                                ▼
   ┌──────────────────────────────────────────────────────────────┐
   │  ③ "쓸 번호 / 뺄 번호" 정하기  (= 채택 번호 풀)               │
   │                                                              │
   │   3가지 분석으로 1~45번 중에서 부적합한 번호를 걸러냅니다.    │
   │                                                              │
   │   ⓐ 편차 분석       (카이제곱 / 평균에서 너무 튄 번호 제외)   │
   │   ⓑ 누적 출현 극값  (2년·전체 각 최다·최소 1개씩 → 고유 합집합 제외)      │
   │   ⓒ 연속 출현 제외  (직전 2회차 연속으로 나온 번호 제외)      │
   │                                                              │
   │   → 위 3가지를 모두 통과한 번호 = "채택 번호 풀"              │
   └────────────────────────────┬─────────────────────────────────┘
                                ▼
   ┌──────────────────────────────────────────────────────────────┐
   │  ④ 조합 규칙으로 6개씩 묶기                                   │
   │                                                              │
   │   채택 번호 풀 안에서만 6개를 골라, 아래 4가지 패턴을          │
   │   동시에 만족하는 세트를 우선순위 순서대로 만듭니다.          │
   │                                                              │
   │   ⓐ 합계 구간   (너무 작거나 너무 큰 합계는 제외)             │
   │   ⓑ 홀짝 비율   (예: 짝수 3개·홀수 3개 같은 흔한 비율)        │
   │   ⓒ 연속 길이   (3연속 같은 흔한 연속 패턴)                   │
   │   ⓓ 자리별 번호대                                            │
   │       (1번 자리는 1~5, 2번 자리는 6~10처럼 자주 나오는       │
   │        구간 패턴을 맞춰서 배치)                              │
   └────────────────────────────┬─────────────────────────────────┘
                                ▼
   ┌──────────────────────────────────────────────────────────────┐
   │  ⑤ 30세트 완성 후 저장                                        │
   │     · 가장 가능성 높은 패턴부터 30세트를 채움                 │
   │     · 같은 번호 조합이 겹치지 않게 다양하게 분산               │
   │     · 결과를 DB에 저장하고 화면에 보여 줌                     │
   └──────────────────────────────────────────────────────────────┘
```

## 2. 30세트는 "이렇게" 만들어집니다

추천 번호 30세트는 **마구잡이 무작위가 아닙니다.**  
크게 **두 단계**로 나뉘어 만들어집니다.

### 1단계 — "후보 번호 풀" 만들기 (3가지 분석)

1~45번 중에서 **이번 회차에 쓸 만한 번호만** 추려 냅니다.

| 분석 | 한 줄 요약 | 어떤 번호가 빠지나? |
|:---|:---|:---|
| **편차 분석** | 과거 평균에서 너무 벗어나는 번호를 솎아냄 | 통계적으로 "지금 흐름"과 안 맞는 번호 |
| **누적 출현 극값** | 본번호만 집계해 2년(104회)·전체 각각 최다 1개·최소 1개를 뽑고, 네 값의 **고유 합**을 제외 후보로 씀(동률 시 번호가 작은 쪽; 슬롯이 겹치면 고유 개수는 4 미만일 수 있음) | 위 규칙으로 정한 번호(추천의 “채택 번호 풀” 계산과 **동일 규칙**) |
| **연속 출현 제외** | 최근 2회차에 모두 등장한 번호를 뺌 | "또 나올 것 같지만 통계적으로 식상한" 번호 |

→ 세 분석을 통과한 번호만 **"채택 번호 풀"** 에 들어갑니다.  
   (보통 1~45 중 20~30개 정도 남습니다.)

### 2단계 — "조합 규칙"으로 6개씩 묶기 (4가지 패턴)

채택 번호 풀 안에서만 6개를 골라, **현실의 당첨번호가 가장 자주 보였던 패턴**을 따라 묶습니다.

| 조합 규칙 | 의미 | 예시 |
|:---|:---|:---|
| **합계 구간** | 6개 번호 총합이 너무 작지도, 크지도 않게 | 21+25+...+38 = 약 100~175 사이 |
| **홀짝 비율** | 짝수와 홀수의 개수 비율을 맞춤 | 짝3·홀3, 짝4·홀2 등 자주 나오는 패턴 |
| **연속 길이** | 이어지는 번호의 최대 길이 | "33-34-35" 같은 3연속이 들어가는 패턴 등 |
| **자리별 번호대** | 6자리 각각이 어느 구간(1~5, 6~10, …) 에 속할지 | 1번 자리는 1~5, 6번 자리는 36~40처럼 자주 보이는 배치. 같은 5단위 구간 안에서는 하·중·상(작·중·큰 번호)이 골고루 쓰이도록 구간 내 칸 사용 빈도를 반영 |

→ 이 4가지 패턴을 **동시에** 만족하는 6개짜리 세트를 만듭니다.

### 3단계 — 우선순위대로 30세트 채우기

- 위 4가지 패턴은 **"가장 자주 나오는 1등 패턴 → 2등 → 3등 …"** 순서가 정해져 있습니다.
- 그 순서대로 한 세트씩 만들면서 **총 30세트**를 채웁니다.
- 이미 만든 세트와 **번호가 너무 겹치지 않도록** 분산해서 고릅니다.
- 30세트가 다 채워지지 않으면 "조건을 살짝 풀어서" 더 만들고, 그래도 부족하면 가능한 만큼만 보여 줍니다.

## 3. 자주 묻는 점

- **왜 30세트인가요?**  
  너무 적으면 선택지가 부족하고, 너무 많으면 패턴이 비슷해져 의미가 줄어듭니다.  
  30세트 정도가 **다양한 패턴을 보여 주면서도 한눈에 비교하기 좋은** 분량입니다.

- **회차를 바꾸면 세트도 바뀌나요?**  
  네. 회차마다 **그 시점까지의 당첨 이력**으로 다시 계산하기 때문에,  
  같은 사람이 다른 회차를 고르면 채택 번호 풀과 30세트가 달라집니다.

- **무작위로 나오는 건가요?**  
  아닙니다. **과거 당첨 통계 → 후보 풀 압축 → 조합 패턴 매칭** 의 3단 필터를 거칩니다.  
  단, 같은 조건에서는 **항상 같은 30세트**가 나오도록 결정됩니다(재현 가능).

- **100% 당첨 보장인가요?**  
  아닙니다. 로또는 본질적으로 확률 게임이며, 이 사이트는  
  **"과거 통계상 더 자주 보였던 패턴"** 을 보여 줄 뿐입니다.

## 4. “누적 번호 분석” 화면이 보여 주는 것

분석 메뉴의 **누적 번호 분석** 페이지는 추천 30세트와 별도로, **한 회차를 골라 조회했을 때** 그 회차 **직전까지** 모아 둔 당첨번호만으로 아래를 한 화면에 보여 줍니다.

1. **전체 누적 막대 차트**  
   선택 회차보다 **이전** 회차들만 사용해, 1~45번 각각이 본번호(보너스 제외)로 **몇 번 나왔는지** 누적 집계합니다.

2. **누적 출현 극값 — 후보 제외**  
   직전 **104회(약 2년)** 구간과, 마찬가지로 선택 회차 **직전까지의 전체** 구간을 각각 잡고, 각 구간에서 본번호만 세어 **최다 1개·최소 1개**를 고릅니다(같은 횟수가 여러 번호면 **번호가 더 작은 쪽**). 이렇게 나온 네 자리를 합쳐 **중복을 없앤 번호 집합**이 “제외 후보”입니다. 네 자리가 같은 번호로 겹치면 고유 개수는 4보다 작을 수 있습니다.  
   이 규칙은 위 **ⓑ 누적 출현 극값**과 **같은 계산**입니다.

3. **2년 구간 상·하 전략 차트**  
   전략 안내 문단 아래에, 직전 104회(2년) 구간 누적만으로 **출현 상위 4개**와 **출현 하위 4개** 전략을 각각 막대 차트와 지표로 보여 줍니다.

4. **전체 기간 상·하와 최종 채택 4개**  
   선택 회차 직전까지의 **전체 기간** 누적 분포에 대해 **출현 상위 4개·하위 4개** 후보를 만든 뒤, 규칙에 따라 한 세트로 합쳐 **최종 채택 4개**를 정합니다(화면에 오름차순으로 표시).  
   2년 창의 상·하는 **같은 규칙**을 쓰되 **2년 구간 전용 표시·전략 후보**이고, 스냅샷에 쓰이는 최종 4개는 **전체 기간 상·하 결합 결과**입니다.

5. **스냅샷 저장 버튼**  
   조회가 끝나고 오류가 없으며, 위 **최종 채택 4개**가 정상적으로 계산된 경우에만 저장(스냅샷)을 누를 수 있습니다.

### 화면 뒤에서 일어나는 일(훅·상태)

- **회차 목록**  
  페이지에 들어오면 서버에서 조회 가능한 회차 번호 목록을 한 번 불러 옵니다. 다른 화면으로 나가면 진행 중이던 요청은 취소됩니다.

- **1회차를 조회한 경우**  
  그보다 이전 회차가 없으므로, 누적 구간·기간 윈도·전략·극값 제외·최종 채택 4개는 계산하지 않고 **해당 회차 당첨번호만** 보여 줍니다.

- **2회차 이상을 조회한 경우**  
  선택 회차 기준 **전 구간 당첨 이력**, **당첨번호(해당 회차)**, **고정 윈도(예: 직전 104회)** 데이터를 **한꺼번에(병렬)** 요청한 뒤, 누적 막대·윈도 차트·극값 제외·전략 결과를 채웁니다.

- **연속으로 조회할 때**  
  조회 버튼을 빠르게 여러 번 누르면, **늦게 끝난 이전 요청**이 나중에 끝난 요청의 결과를 덮어쓰지 않도록 **조회마다 번호(세션)**를 붙여, 오래된 응답은 화면을 바꾸지 않습니다.

- **스냅샷 저장 조건**  
  **이미 조회가 끝난 회차 문자열**이 있고, 조회 중이 아니며, 조회 오류가 없고, 그 회차가 **2 이상**이며, **최종 채택 4개**가 모였을 때만 저장을 허용합니다.  
  저장 시 **2 미만**이면 「회차 2 이상 조회 후에만…」, **4개가 아직 없으면** 「최종 채택 4개 번호가 계산된 뒤에만…」 같은 안내로 막습니다.

- **상단 안내 문구**  
  아래 순서로 **가장 먼저** 걸리는 것만 보여 줍니다: 회차 목록 로딩 중 → 목록 오류 → 목록이 비었음 → 조회 집계 중 → 조회 오류 → (이미 조회 완료면 문구 없음) → 아직 조회 전이면 조회 방법 안내.

- **추천 30세트 화면과의 차이**  
  추천 기능은 채택 번호 풀과 조합 규칙으로 **30세트**를 만들고, 누적 번호 분석은 한 회차 조회 결과로 **2년·전체 상·하 전략과 최종 4개**만 보여 주는 **별도 화면**입니다. 같은 출현 집계나 전략 평가 함수를 일부 공유할 수 있어도, 위 §1~3의 “30세트가 만들어지는 흐름”과 **동일하지 않습니다**.
