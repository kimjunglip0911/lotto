# 로또 번호 추천 로직

`frontend/src/app/recommend` 페이지에서 번호 세트가 추천되는 전체 흐름을 정리합니다.

---

## 전체 흐름 요약

```
[사용자] 생성 및 저장 실행 버튼 클릭
    ↓
[프론트엔드] GET /api/recommend/exclusion-candidates
    ↓
[프론트엔드] 제외 규칙 파이프라인 실행 (로컬)
    ↓
[프론트엔드] POST /api/recommend/generate-and-save
    ↓
[화면] 적용 규칙 / 제외 번호 / 생성된 20개 세트 표시
```

---

## 1단계: 제외 후보 조회 (`GET /api/recommend/exclusion-candidates`)

### 대상 회차 결정

- `draw_no` 쿼리 파라미터가 없으면, `lotto_winners` 테이블의 최대 `draw_no + 1`을 대상 회차로 사용합니다.

### 기간별 당첨 번호 집계

대상 회차보다 이전인 당첨 번호를 6개 기간 윈도우별로 조회합니다.

| 윈도우 키 | 기간 | 조회 행 수 (회차 기준) |
|---|---|---|
| `overall` | 전체 | 제한 없음 |
| `sixMonth` | 6개월 | 최근 26회 |
| `oneYear` | 1년 | 최근 52회 |
| `threeYear` | 3년 | 최근 156회 |
| `fiveYear` | 5년 | 최근 260회 |
| `tenYear` | 10년 | 최근 520회 |

각 기간에서 번호별 출현 횟수를 집계합니다. 집계 대상 필드는 `num1~num6`, `bonus_num` (7개) 모두 포함됩니다.

### 동점(tie) 처리 정책

- 최고 출현 번호 선정 시: 동점이면 **가장 작은 번호** 선택 (`min-number-on-tie`)

### 응답 데이터

| 필드 | 설명 |
|---|---|
| `drawNo` | 대상 회차 |
| `leastFrequentOverall` | 전체 누적 기준 최저 출현 번호 정보 (참고용, 제외에 사용하지 않음) |
| `windowTopNumbers` | 6개 기간별 최고 출현 번호 정보 |
| `excludedNumbersUnion` | 6개 기간별 1등 번호의 합집합(중복 제거, 오름차순) |
| `drawCounts` | 기간별 조회된 회차 수 |
| `ruleMeta` | 동점 처리 정책 및 집계 필드 정보 |

---

## 2단계: 제외 규칙 파이프라인 (프론트엔드 로컬 실행)

`runRecommendPipeline` 함수가 규칙 배열을 순서대로 실행하며 제외 번호를 누적합니다.

각 규칙은 `{ exclusionCandidates, currentExcludedNumbers }` 컨텍스트를 받아 제외할 번호 목록을 반환하며, 다음 규칙 실행 시 이전 규칙의 결과가 `currentExcludedNumbers`에 포함됩니다.

### 규칙 1: 기간별 1등 번호 집합 제외 (`exclude-top-rank-from-windows`)

- `windowTopNumbers`의 6개 기간 각각에서 최고 출현 번호의 **전체 후보(`candidates`)** 를 수집합니다.
- 동점(tie)인 경우에도 해당 출현 횟수를 공유하는 모든 번호를 빠짐없이 제외합니다.
- 중복 제거 없이 6개 기간의 후보 배열을 그대로 합쳐 오름차순 정렬합니다.
- 이유 예시: `"전체 [X, Y]번, 6개월 [Z]번, 1년 [A, B]번, ... 을 집합으로 합쳐 제외합니다."`

### 최종 제외 번호

규칙 실행 결과를 합친 집합(중복 제거, 오름차순 정렬)이 최종 `excludedNumbers`가 됩니다.

## 3단계: 번호 세트 생성 및 저장 (`POST /api/recommend/generate-and-save`)

### 요청 바디

| 필드 | 설명 |
|---|---|
| `draw_no` | 대상 회차 |
| `applied_rule_ids` | 적용된 규칙 ID 목록 |
| `excluded_numbers` | 최종 제외 번호 목록 |

### 번호 세트 생성

- **JL Wheel Method** (`generate_jl_wheel_sets`)로 20개 세트를 생성합니다.
- `features` 모듈이 없는 환경에서는 폴백으로 1~45에서 무작위 6개를 추출해 20개 세트를 생성합니다.

> **참고**: 현재 백엔드는 `excluded_numbers`를 JL Wheel 생성 함수에 직접 전달하지 않습니다. 제외 번호는 결과 세트의 메타 정보(`excluded_numbers` 필드)로만 기록됩니다.

### DB 저장

1. 동일 `draw_no` + `method = "JL Wheel Method"` 인 기존 레코드를 `lotto_drawings` 테이블에서 삭제합니다.
2. 20개 세트를 각각 `lotto_drawings`에 INSERT합니다.
   - `group_id`: `jl_` + 12자리 UUID hex
   - `method`: `"JL Wheel Method"`
   - `bonus_num`, `draw_count`: 0 (미사용)

### 응답

저장된 20개 세트를 반환합니다. 각 세트에 `applied_rule_ids`, `excluded_numbers` 메타 정보가 포함됩니다.

|---|---|
| `page.tsx` | 전체 오케스트레이션 (API 호출, 파이프라인 실행, 상태 관리) |

| `logic/pipeline.ts` | 규칙 배열을 순회하며 제외 번호 누적 계산 |
| `logic/rules/excludeTopRankFromWindows.ts` | 기간별 최고 출현 번호 집합 제외 |
| `logic/types.ts` | 공통 타입 정의 |
