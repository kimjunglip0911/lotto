# 로또 앱 동작 로직 (안내)

과거 당첨 데이터를 어떻게 불러오고, 화면에 어떻게 보여 주고, 저장하는지 **큰 줄기**만 정리합니다. 코드나 함수 이름은 적지 않습니다.

---

## 1. 전체 로직

### 1.1 이 앱이 하는 일 한눈에

| 구분 | 내용 |
|:---|:---|
| **데이터** | 과거 당첨 번호(본번호 6개, 보너스 1개)가 저장되어 있음 |
| **역할 나누기** | **서버**: 당첨 이력 조회, 번호 세트 만들기, DB 저장 **사용자 기기(브라우저)**: 제외 규칙 적용, 일부 분석 계산, 화면 표시 |
| **메뉴별 관계** | **추천**, **누적 번호 분석**, **연속 미출현 분석**은 각각 별도 화면이며, 필요할 때만 서로 같은 생각 방식을 공유함 |

### 1.2 공통으로 알아두면 좋은 것

- **본번호 6개**와 **보너스 1개**를 합쳐 7개로 세는 경우(출현 횟수 등)와, **본번호 6개만** 비교하는 경우(“몇 개 맞췄나” 같은 평가)가 **화면마다 다름**. 아래 각 기능에서 따로 적음.
- **동점**(같은 횟수로 1등이 여러 번호일 때)은 기능마다 처리 방식이 조금 다름. 추천의 “제외 후보 조회”는 작은 번호 하나만 쓰고, 추천의 “규칙 1”은 동점인 번호를 **모두** 후보에 넣음.

---

## 2. 각 분석·기능별 로직

### 2.1 추천 화면 (번호 세트 만들기)

**목적:** 정해진 규칙으로 “빼고 싶은 번호”를 정한 뒤, 6개짜리 번호 세트 **20개**를 만들어 저장한다.

#### 흐름 (순서대로)

1. **제외 후보 받기 (서버)**  
   - 대상 회차: 사용자가 안 고르면 “다음에 나올 회차”처럼 **가장 최근 당첨 다음 회차**를 쓴다.  
   - 전체·6개월·1년·3년·5년·10년 같은 **여러 기간**마다, 그 안에서 **가장 많이 나온 번호**를 구한다.  
   - 기간마다 1등 번호들을 모으면 **합집합**(중복 없이)이 되고, 이게 “제외 후보”에 가까운 정보다.  
   - **전체 기간에서 가장 적게 나온 번호** 정보는 참고용으로만 주고, 제외 계산에는 쓰지 않는다.

2. **제외 규칙 적용 (사용자 기기)**  
   - 화면에 켜 둔 규칙을 **정해진 순서**로 하나씩 적용한다.  
   - **규칙 1:** 기간마다 “가장 많이 나온 번호”를 고를 때 동점이면 **해당 번호 전부**를 후보에 넣는다. 여섯 기간 결과를 이어 붙여 정리한다.  
   - 여기까지 나온 번호들을 합치면 **최종 제외 번호 목록**이 된다.

3. **생성·저장 (서버)**  
   - 대상 회차, 적용한 규칙 목록, 최종 제외 번호를 보낸다.  
   - 서버가 정해진 방식으로 **20개** 세트를 만든다. (환경에 따라 단순 무작위로 채울 수도 있음.)  
   - **제외 목록은 “번호를 뽑는 수식”에 직접 넣지 않고**, 저장 결과에 “이번에 이런 번호를 빼고 생각했다”는 기록으로 남긴다.  
   - 같은 회차·같은 방식으로 예전에 저장한 것이 있으면 지우고, 새 20개를 DB에 넣는다.

4. **화면**  
   실행 진행 안내, 적용 규칙, 제외된 번호, 만들어진 20개 세트를 한곳에 보여 준다.

### 2.2 누적 번호 분석 (별도 메뉴)

**목적:** 사용자가 **기준 회차**를 정하면, 그 회차 **이전까지** 당첨 데이터만으로 “1~45번이 각각 몇 번 나왔는지”와 전략별 차트·추천 4개를 본다.

#### 흐름 (순서대로)

1. **데이터 받기**  
   기준 회차 **이전** 전체 이력과, 선택한 회차 **당첨 1건**을 서버에서 받는다.

2. **누적 막대 차트(전체)**  
   받은 이력 전체로 1~45번 출현 횟수를 센다.

3. **과거 성적표**  
   과거로 거슬러 올라가며, “직전 N회” 길이를 여러 가지로 바꿔 가며 **두 가지 방식**으로 “그때 4개를 골랐다면 본번호 6개와 몇 개 맞았을지”를 쌓는다.  
   - **방식 A:** 평균 출현에 가깝게 4개 고르기  
   - **방식 B:** 많이 나온 번호 2개 + 적게 나온 번호 2개  

4. **차트에 쓸 기간(N) 고르기**  
   성적표를 보고 **단기용 “직전 N회” 2개**, **장기용 “직전 N회” 2개**를 고른다. 비슷한 길이만 몰리지 않게 간격을 두고, 장기는 충분히 긴 N이 되도록 조건을 둔다. 후보가 부족하면 순위로 채운다.

5. **전략 막대 차트 4개 + 전략별 추천 4개**  
   위에서 고른 N만큼 잘라 막대 차트를 네 개 그린다. **N은 조회할 때마다 달라질 수 있다.**  
   전략마다 추천 4개를 만든 뒤, **같은 전략 안**에서 서로 다른 두 기간의 점수는 **합쳐서** 하나로 묶는다.

6. **최종 4개**  
   두 전략 결과를 합친다. **두 전략에 공통으로 나온 번호를 먼저** 넣고, 남는 칸은 점수가 높은 쪽을 우선한다. 가능하면 홀·짝 균형, 연속 번호 줄이기, 끝자리 겹침 줄이기, 낮은·중간·높은 번호대 분산을 맞추고, 안 되면 점수 위주로 채운다.

#### 집계와 “맞췄다” 기준

| 구분 | 무엇을 세고 비교하나요 |
|:---|:---|
| 차트·전략 안 출현 횟수 | 회차마다 **본번호 6 + 보너스 1**(7개) |
| 과거 “몇 개 맞췄나” | 예측 4개 vs **본번호 6개만** (보너스 제외) |

#### 스냅샷 저장 조건 (모두 만족할 때만)

- 조회가 끝났고 오류 없음  
- 기준 회차가 **2회 이상**  
- 화면에 **최종 4개**가 모두 나옴  

#### 추천 화면과의 연결

추천 쪽에서 “당첨 이력만으로 뽑은 4개”가 필요할 때, 이 분석과 **같은 규칙**으로 기간을 고르고 번호를 합친다. **이 화면만** 써도 위 과정은 그대로이며, 추천 메뉴를 열 필요는 없다.

### 2.3 연속 미출현 분석 (별도 메뉴)

**목적:** 1~45번 각각이 **마지막으로 당첨된 뒤 몇 회차 동안 안 나왔는지**를 보여 준다. 통계적 착각을 줄이려는 **안내 문구**가 함께 있다.

#### 흐름 (순서대로)

1. **회차 목록 (서버, 처음 한 번)**  
   DB의 `lotto_winners`에 저장된 회차 번호로 선택 상자를 채운다. **한 건도 없으면** 목록이 비어 있고, 화면에 “당첨 이력이 없으니 홈에서 먼저 저장하라”는 식의 안내가 나온다(새 DB이거나 아직 당첨번호를 넣지 않은 경우).

2. **선택 회차 상세 (서버)**  
   - 선택 회차의 **당첨 번호**(차트에서 강조용)  
   - 그 회차 **이전**까지의 당첨 이력 목록  
   - **1회**만 선택하면 이전 이력이 없어 “집계할 데이터가 없습니다” 안내만 한다.

3. **연속 미출현 계산 (사용자 기기)**  
   이력을 순서대로 보며 **본번호 6 + 보너스 1**을 “나온 번호”로 보고, 각 번호의 **마지막으로 나온 회차**를 기록한다.  
   - **연속 미출현** = 선택 회차 번호 − 마지막 출현 회차  
   - 한 번도 안 나왔으면 선택 회차 번호 그대로가 된다.

4. **요약 지표**  
   - 45개 번호 연속 미출현의 **평균**  
   - **가장 긴** 연속 미출현  
   - 길이를 작은 순으로 줄 세웠을 때 **상위 5%**에 해당하는 경계값  
   - **저빈도 후보:** 평균보다 길게 안 나온 번호(평균이 0보다 클 때만 의미 있음)

#### 화면 구성

- **요약 카드:** 분석 회차 수, 최장 연속 미출현, 평균, 상위 5% 기준, 저빈도 후보 개수  
- **막대 차트:** 1~45번 연속 미출현. 당첨 번호는 노란색, 저빈도 후보는 주황색, 나머지는 보라색. 상위 5% 경계에는 점선  
- **저빈도 후보 목록:** 평균 초과인 것만, 긴 순으로 정렬  
- **표:** 번호 / 마지막 출현 회차 / 연속 미출현 / 판정  

#### 추천·최종 4개 화면과의 연결

**선택 회차의 바로 이전 두 회차**의 본번호만 본다. 그 두 회차에 **공통으로 나온 본번호**를 모으면, 추천 화면과 “최종 4개” 화면에서 **제외 후보**로 가져다 쓴다. 보너스는 여기 비교에 넣지 않는다.
