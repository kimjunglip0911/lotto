# 로또 번호 추천 로직

**추천 화면**에서 번호 세트가 만들어지고 저장되기까지의 흐름을, 사용자가 이해하기 쉬운 말로 정리합니다.

---

## 전체 흐름 요약

```
[사용자] 생성 및 저장 실행 버튼 클릭
    ↓
[프론트엔드] GET /api/recommend/exclusion-candidates
    ↓
[프론트엔드] 제외 규칙을 순서대로 적용 (로컬)
    ↓
[프론트엔드] POST /api/recommend/generate-and-save
    ↓
[화면] 적용 규칙 / 제외 번호 / 생성된 20개 세트 표시
```

---

## 1단계: 제외 후보 조회 (`GET /api/recommend/exclusion-candidates`)

### 대상 회차 결정

- `draw_no` 쿼리 파라미터가 없으면, `lotto_winners` 테이블의 최대 `draw_no + 1`을 대상 회차로 사용합니다.

### 기간별 당첨 번호 집계

대상 회차보다 이전인 당첨 번호를 6개 기간 윈도우별로 조회합니다.

| 윈도우 키 | 기간 | 조회 행 수 (회차 기준) |
|---|---|---|
| `overall` | 전체 | 제한 없음 |
| `sixMonth` | 6개월 | 최근 26회 |
| `oneYear` | 1년 | 최근 52회 |
| `threeYear` | 3년 | 최근 156회 |
| `fiveYear` | 5년 | 최근 260회 |
| `tenYear` | 10년 | 최근 520회 |

각 기간에서 번호별 출현 횟수를 집계합니다. 집계 대상 필드는 `num1~num6`, `bonus_num` (7개) 모두 포함됩니다.

### 동점(tie) 처리 정책

- 최고 출현 번호 선정 시: 동점이면 **가장 작은 번호** 선택 (`min-number-on-tie`)

### 응답 데이터

| 필드 | 설명 |
|---|---|
| `drawNo` | 대상 회차 |
| `leastFrequentOverall` | 전체 누적 기준 최저 출현 번호 정보 (참고용, 제외에 사용하지 않음) |
| `windowTopNumbers` | 6개 기간별 최고 출현 번호 정보 |
| `excludedNumbersUnion` | 6개 기간별 1등 번호의 합집합(중복 제거, 오름차순) |
| `drawCounts` | 기간별 조회된 회차 수 |
| `ruleMeta` | 동점 처리 정책 및 집계 필드 정보 |

---

## 2단계: 제외 규칙 적용 (사용자 기기에서 계산)

미리 받아 둔 **제외 후보**와 **지금까지 제외된 번호**를 바탕으로, 화면에 설정된 규칙을 **정해진 순서대로** 하나씩 적용합니다. 각 규칙이 “이번에 빼야 할 번호”를 더해 나가고, 다음 규칙은 그 결과를 이어받습니다.

### 규칙 1: 기간별로 가장 많이 나온 번호 묶음 제외

- 6개 기간 각각에서 “가장 많이 나온 번호”를 고를 때, **같은 횟수로 동률이면 그 번호들을 모두** 후보에 넣습니다.
- 중복 제거 없이 6개 기간의 후보 배열을 그대로 합쳐 오름차순 정렬합니다.
- 이유 예시: `"전체 [X, Y]번, 6개월 [Z]번, 1년 [A, B]번, ... 을 집합으로 합쳐 제외합니다."`

### 최종 제외 번호

규칙 실행 결과를 합친 집합(중복 제거, 오름차순 정렬)이 최종 제외 번호 목록이 됩니다.

---

## 3단계: 번호 세트 생성 및 저장 (`POST /api/recommend/generate-and-save`)

### 요청 바디

| 필드 | 설명 |
|---|---|
| `draw_no` | 대상 회차 |
| `applied_rule_ids` | 적용된 규칙 ID 목록 |
| `excluded_numbers` | 최종 제외 번호 목록 |

### 번호 세트 생성

- 서버는 정해진 **번호 조합 생성 방식**으로 6개 번호가 들어간 세트를 **20개** 만듭니다.
- 일부 환경에서는 대신 1~45 사이에서 무작위로 6개를 뽑는 방식으로 채울 수 있습니다.

> **참고**: 제외 번호 목록은 **번호를 만드는 단계에 직접 넘기지 않고**, 저장되는 결과에 “어떤 번호를 빼고 계산했는지”만 함께 적어 두는 형태입니다.

### DB 저장

1. 동일 `draw_no` + `method = "JL Wheel Method"` 인 기존 레코드를 `lotto_drawings` 테이블에서 삭제합니다.
2. 20개 세트를 각각 `lotto_drawings`에 INSERT합니다.
   - `group_id`: `jl_` + 12자리 UUID hex
   - `method`: `"JL Wheel Method"`
   - `bonus_num`, `draw_count`: 0 (미사용)

### 응답

저장된 20개 세트를 반환합니다. 각 세트에 `applied_rule_ids`, `excluded_numbers` 메타 정보가 포함됩니다.

---

## 화면에서 하는 일 (요약)

- **실행** 버튼과 진행 안내를 보여 주고, 적용된 규칙·제외된 번호·생성된 20개 세트를 한곳에 모아 보여 줍니다.

---

# 누적 번호 분석

추천 화면과는 **별도 메뉴**입니다. 사용자가 **기준 회차**를 정하고 조회하면, 그 회차 **이전**까지의 당첨 번호만으로 다음을 보여 줍니다.

- 1~45번 각각이 지금까지 몇 번 나왔는지(누적 출현)
- **평균근접 2개 N**과 **상2+하2 2개 N**으로 잘라 그린 막대 차트(네 개 N은 조회할 때마다 달라질 수 있음)
- 두 전략을 섞어 낸 **최종 추천 4개**

## 처리 순서

1. **조회**  
   서버에서 기준 회차 **이전** 당첨 이력 전체와, 선택한 회차의 당첨 1건을 받아 옵니다.

2. **전체 누적 차트**  
   조회한 이력 **전체**로 1~45번 출현 횟수를 셉니다.

3. **과거 성적표 만들기**  
   과거 회차를 거슬러 올라가며, 여러 “직전 N회” 길이와 **두 가지 전략**으로 “그때 4개를 골랐다면 본번호 6개와 몇 개 맞았을지”를 쌓습니다.  
   - 한 전략은 **평균 출현에 가깝게** 4개를 고릅니다.  
   - 다른 전략은 **많이 나온 번호 2개 + 적게 나온 번호 2개**를 고릅니다.

4. **단기·장기에 쓸 N 고르기**  
   위 성적표를 보고 **단기용 직전 N회 2개**, **장기용 직전 N회 2개**를 고릅니다. 비슷한 길이만 몰리지 않게 간격을 두고, 장기 쪽은 N이 충분히 길도록 조건을 둡니다. 후보가 부족하면 순위로 채웁니다.

5. **전략 막대 차트(4개)와 전략별 추천 4개**  
   4단계에서 고른 **평균근접용 N 2개**, **상2+하2용 N 2개**만큼 잘라 막대 차트를 그립니다(차트에 쓰인 N은 조회마다 달라질 수 있음). 전략마다 추천 4개를 만든 뒤, **같은 전략 안**에서 서로 다른 두 기간의 점수는 **합산**해 하나로 묶습니다.

6. **최종 4개 정하기**  
   두 전략의 추천을 합칩니다. **두 전략에 공통으로 나온 번호를 먼저** 넣고, 남는 칸은 점수가 높은 쪽을 우선합니다. 가능하면 홀·짝 균형, 연속 번호 남발 완화, 끝자리 중복 완화, 낮은·중간·높은 번호대 분산을 맞추고, 맞출 수 없을 때만 점수만으로 채웁니다.

## 집계할 때와 “맞췄다”고 볼 때

| 구분 | 무엇을 세고 비교하나요 |
|:---|:---|
| 차트·전략 안의 출현 횟수 | 회차마다 **본번호 6개 + 보너스 1개**(총 7개)를 셉니다. |
| 과거 성적(적중 개수) | 예측 4개와 **본번호 6개만** 비교합니다. 보너스는 넣지 않습니다. |

## 막대 차트 기간은 고정인가요

**아니요.** 화면에 나오는 전략 막대 차트 네 개는 각각 **직전 N회** 한 덩어리이고, N은 다음처럼 나뉩니다.

- **평균근접(단기 쪽)**에서 쓸 N **2개**
- **상2+하2(장기 쪽)**에서 쓸 N **2개**

이 네 값은 성적표를 보고 자동으로 고르기 때문에, **기준 회차나 과거 데이터가 바뀌면 차트에 쓰인 N도 함께 바뀔 수 있습니다.**  
(내부 평가에서는 여러 N 후보를 두루 훑은 뒤, 위 네 구간만 화면에 씁니다.)

## 스냅샷을 저장할 수 있는 조건

아래를 **모두** 만족할 때만 저장할 수 있습니다.

- 조회가 끝났고 오류가 없음
- 기준 회차가 **2회 이상**
- 화면에 **최종 4개**가 모두 계산됨

## 추천 기능과의 연결

추천 기능에서 “당첨 이력만으로 뽑은 4개”가 필요할 때, 이 분석과 **같은 규칙**으로 기간을 고르고 번호를 합칩니다. **누적 번호 분석 화면만** 써도 위 과정은 그대로이며, 추천 메뉴를 열 필요는 없습니다.
