**[중요]**
만약 1등,2등,3등이 나온다면 무조껀 좋은것임(5등이 100개 나와도 무의미)

**[요청사항]**
1. 5등 당첨이 1번 된 세트 더 좋은 결과가 나오도록 속도 재조정.
2. 1번 테스트 이후 기존보다 더 안좋은 수치가 나올시 해당 세트만 속도 재조정하여 다시 진행
3. 다시 진행 할떄는 기존과는 다른 각도로 생각하여 속도 재조정.(--tune-delta 수정 등)
4. **시작 번호(6개) 선정**  
   - 기준 구간: **1회차 ~ 작업 시점의 현재 회차**까지 공개된 당첨 데이터.  
   - 각 로또 번호(1~45)에 대해 위 구간에서의 **당첨 누적 합산**(번호별 누적치)을 계산한다.  
   - **당첨 누적 합산이 가장 큰 6개 번호**를 한 조합으로 정해, 생성·시뮬의 **시작(고정) 번호**로 사용한다.  
   - 6위권에 동점이 있으면 **번호 오름차순** 등으로 순위를 매겨 6개를 확정한다 (구현 시 코드·주석에 타이브레이크를 명시).
5. **속도 프로파일 20종 적용**  
   - **세트# 1~20**에 대해 서로 다른 **초기 속도(speed)** 를 둔다 (한 세트당 하나의 기준 속도).  
   - 동일한 시작 번호·동일한 평가 구간에서 세트 간에는 **속도만** 바꾸고, 산출물은 `backend/docs/당첨 이력.md` 고정 양식으로 정리한다.
6. **정지 시간 고정(공정한 비교)**  
   - `stop_time = speed / deceleration` 인 물리식을 쓰는 한, **비교 실험에서는 정지 시간을 한 상수로 고정**한다.  
   - 세트마다 바꾸는 것은 **speed(초기 속도) 20단계**뿐이며, 감속도는 **`deceleration = speed / 고정정지시간`** 으로 **파생**시켜 항상 같은 정지 시간이 나오게 한다.  
   - 중복 번호로 인한 **재시도·지터로 speed를 흔들 때도** 동일 규칙을 적용한다 (그 시점의 speed에 맞춰 decel을 다시 계산).  
   - 고정값은 `backend/docs/당첨 이력.md` 등 과거 결과 중 **가장 유리했던(예: 4등 포함 등 상위 등수) 세트**의 `speed÷decel`로 정하고, 코드 상수(`jl_service.FIXED_STOP_TIME`)와 주석에 출처를 남긴다.
7. **동일 세트 처리 규칙 (사후 교체 방식)**  
   - 동일 세트가 나오더라도 생성 단계에서 시작 번호를 시프트해 재생성하지 않고, **먼저 20세트를 모두 생성**한다.  
   - 그 후 기준 구간(기본: `1회차 ~ 현재 회차`)의 번호별 **누적 당첨 횟수**를 계산하고, 누적이 가장 낮은 번호 **20개를 교체 후보 풀**로 만든다(동률 시 번호 오름차순).  
   - **세트#1은 유지**하고, 세트#2부터 이전 세트들과 비교해 동일 조합이 발견되면 해당 세트 6개 중 **누적 당첨 횟수가 가장 많은 번호 1개**를 교체한다.  
   - 교체 번호는 후보 풀에서 **낮은 번호부터 순차 사용**하며, 교체 후에도 중복이면 후보 풀의 다음 번호로 반복 교체한다.  
   - 예: 세트#2가 세트#1과 동일이면 세트#2를 교체하고, 이후 세트#5가 세트#2와 동일해지면 세트#5를 다시 같은 규칙으로 교체한다.

### `backend/docs/당첨 이력.md` 출력 규칙 (고정 양식)

이 파일은 **항상 전체 덮어쓰기**하며, 아래 구조·표기를 **변경하지 않는다**.

1. **제목 (2단계 헤더)**  
   `## 2. 속도 프로파일(세트#)별 당첨 이력`

2. **요약 블록** (헤더 다음 빈 줄 1줄 후, 등수별 집계)  
   - `당첨 누적 회차 : {숫자}회` — 표의 **회차(등수)** 열에 등장하는 **서로 다른 회차 번호** 개수  
   - `1등 : {숫자}번` ~ `5등 : {숫자}번` — 표 전체에서 해당 등수 출현 횟수 (0이어도 `0번`으로 명시)

3. **빈 줄 1줄** 후 **마크다운 표**

4. **표 형식**  
   - 헤더 행: `| 세트# | speed | decel | 회차(등수) |`  
   - 구분 행: `|-------|-------|-------|------------|`  
   - 데이터 행: 세트 번호 1~20, 각 행에 `speed`, `decel`(소수 가능), 마지막 열은 `회차번호(등수)`를 **쉼표+공백**으로 연결 (예: `1172(5등), 1199(5등)`). 당첨이 없으면 해당 칸은 비우거나 `-` 등 **일관된 단일 규칙**으로만 표기.

5. **집계 일관성** — 요약의 `4등`·`5등`·`당첨 누적 회차`는 표 내용과 **반드시 일치**하도록 갱신한다.

스크립트·에이전트·수동 편집 모두 위 양식을 준수해 **파일 전체를 한 번에 교체**한다 (부분 패치로 형식이 깨지지 않게 할 것).
