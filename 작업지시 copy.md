# 각 기법 고도화 작업 지시서

## 사용법 (내일 진행 시)

**이 파일을 작업 지시로 던져주면**, 아래 "참고·고도화 기법" 표에서 **고도화가 아직 완료되지 않은 10개 기법 모두**를 대상으로 고도화를 진행한다.  
표에서 "고도화 완료"가 비어 있거나 X인 기법은 모두 고도화 대상으로 간주하고, **기법별로 회차당 1세트만 생성**하는 전용 경로로만 작업한다.  
**20세트 생성은 사용하지 않는다.**

**사용 스크립트 (2개만 사용)**  
- **기법 고도화**: `python -m scripts.run_technique_52 --method <기법명>`  
  (기법명: order_statistics | frequency_trend | cdm | markov | lstm | bilstm | cnn | ga | pso | behavioral)  
- **융합**: `python -m scripts.run_fusion_52` (생성+분석) / `python -m scripts.run_fusion_52 --tune` (가중치 튜닝 포함)

---

## 1. 목표

- **최종 목표**: 최근 52회(1164~1215) 중 **26회 이상**에서 융합 20세트 중 1세트라도 5등 이상.
- **전략**: 각 기법을 **52회·회차당 1세트·4등 합격 기준**으로 개별 고도화한 뒤, **융합 가중치만** 탐색하여 조합 효과를 극대화한다.

---

## 2. 고도화 합격 기준 (필수)

| 항목 | 내용 |
|------|------|
| **생성 방식** | **해당 기법만** 회차별 **1세트씩만** 생성. 52회 × 1세트. |
| **금지 사항** | 고도화 작업 중 **20세트 생성 절대 사용 금지**. 기법 고도화는 반드시 `run_technique_52`만 사용한다. |
| **합격 기준** | 52회차 중 **5회 이상**에서 그 1세트가 **4등**(당첨번호 4개 일치) 이상이면 해당 기법 고도화 **합격**으로 간주한다. |
| **평가** | 회차당 1세트만 생성한 뒤, 당첨번호와 비교하여 4개 이상 일치 회차 수를 센다. 5회 이상이면 합격, 미만이면 상수 재조정 후 재시도. |

- **정리**: 고도화 진행 시 **기법당 52회 × 1세트만** 생성하고, **52회 중 5회 이상 4등**이면 고도화 완료.

---

## 3. 실행 순서 및 방법

### 3.1 전체 흐름

```
[1단계] 고도화 미완료 기법 확인 (아래 "참고·고도화 기법" 표 기준)
    ↓
[2단계] 기법별로 `run_technique_52 --method <기법명>` 실행 (52회 1세트 생성 + 분석)
    ↓
[3단계] 52회 중 4등 이상 5회 미만이면 상수 그리드/튜닝 후 재생성·재분석 반복
        5회 이상이면 해당 기법 고도화 완료 처리, 다음 기법으로
    ↓
[4단계] 10개 기법 모두 위 기준으로 고도화 완료 후, `run_fusion_52 --tune` 실행 (융합 가중치 탐색)
```

### 3.2 기법별 고도화 절차 (한 기법당, 1세트만 생성)

1. **대상 기법의 서비스 파일**  
   `backend/domain/services/analysis/` 아래 해당 `*_service.py` 확인.
2. **실행**  
   `python -m scripts.run_technique_52 --method <기법명>`  
   한 번에 해당 기법 52회×1세트 생성 + 4등 이상 회차 수 분석. (기법명은 표의 서비스명과 동일: order_statistics, frequency_trend, cdm, markov, lstm, bilstm, cnn, ga, pso, behavioral)
3. **분석**  
   생성된 52회×1세트에 대해 당첨번호와 비교, **4등(4개 일치) 이상**인 회차 수를 센다. 52회 중 **5회 이상**이면 합격.
4. **조정 가능 상수**  
   파일 상단 상수 블록 또는 README 표 참고. 합격할 때까지 그리드/튜닝으로 상수 조합을 바꿔가며 2~3 반복.
5. **합격 시**  
   최종 상수를 해당 서비스 파일에 반영하고, `backend/domain/services/analysis/README.md`에 적용값·결과(52회 중 4등 N회) 기록. 표에 해당 기법 "고도화 완료" 표시.

### 3.3 융합 단계 (10개 기법 모두 고도화 완료 후)

- **실행**: `cd backend` 후 `python -m scripts.run_fusion_52 --tune`
- **역할**: 융합 52회 생성 + 분석 후, **융합 가중치(WEIGHTS)** 그리드 탐색.
- **결과**: 최고 조합이 `fusion_service.WEIGHTS`에 저장됨. 로그는 `backend/tune_fusion_52_grid.log`.

### 3.4 주의사항

- **고도화 작업 중에는 20세트를 만들지 않는다.** 회차당 1세트만 생성하는 경로만 사용한다.
- 기법별로 순차 고도화. 한 기법 합격 후 다음 기법 진행.

## 4. 참고·고도화 기법 (고도화할 기법만 칸에 표시)

**설명**: 아래 표에서 **고도화할 기법**만 "고도화 여부" 칸에 `O` 또는 `✓`를 넣는다.  
**이 파일을 지시로 던졌을 때**에는 "고도화 완료"가 비어 있는 기법을 **모두** 고도화 대상으로 보고, 위 합격 기준(52회×1세트, 5회 이상 4등)으로 순차 진행한다.

| 번호 | 기법명 (서비스 파일) | 고도화 여부 (O/X) | 고도화 완료 (O/X) | 비고 (상수·스크립트 등) |
|------|----------------------|-------------------|-------------------|--------------------------|
| 1 | 순서 통계량 (order_statistics) |  |  | run_technique_52 --method order_statistics |
| 2 | 출현 빈도·추세 (frequency_trend) |  |  | run_technique_52 --method frequency_trend |
| 3 | CDM 바이시안 (cdm) |  |  | run_technique_52 --method cdm |
| 4 | 마르코프 (markov) |  |  | run_technique_52 --method markov |
| 5 | LSTM (lstm) |  |  | run_technique_52 --method lstm |
| 6 | BiLSTM (bilstm) |  |  | run_technique_52 --method bilstm |
| 7 | CNN (cnn) |  |  | run_technique_52 --method cnn |
| 8 | GA (ga) |  |  | run_technique_52 --method ga |
| 9 | PSO (pso) |  |  | run_technique_52 --method pso |
| 10 | 행동 경제학 (behavioral) |  |  | run_technique_52 --method behavioral |

- **고도화 여부**: 이번에 고도화를 진행할 기법만 O/✓.
- **고도화 완료**: 합격 기준(52회 중 5회 이상 4등)을 달성한 뒤 O/✓ 표시.
