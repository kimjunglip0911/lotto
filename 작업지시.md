## 연구/개선 프레임 요약(압축)

> 목표: 현재 시스템에서 **3등 이상**을 만들기 위한 데이터 기반 개선 루프.
> 원칙: **상위 등수(1~4 또는 1~3) 악화는 0% 허용(즉시 기각)**.

### 분석(필수 4종)
- **B-1 재출현율**: 전회차→이번회차 겹치는 번호 개수 분포(전회차 번호 포함 비율 판단)
- **B-2 핫/콜드**: 최근 10/20/30회 윈도우 핫·콜드 추출(시작번호 후보 비교)
- **C-1 합계 분포**: 합계 최빈 구간 확인(필터 기준 후보 도출)
- **E-3 다중 시드**: seed 의존성 점검(1차 1~20 → 최종 1~100)

### 개선안(D-계열) 우선순위(적용은 1개씩)
- **D-1 시작번호 블렌딩**(직전회차 + 핫 + overdue 가중)
- **D-3 조합 필터**(합계/홀짝/고저/끝자리/동반출현 등은 “게이트 통과 후” 단계 적용)
- **D-4 교체 풀 고도화**(중복 해소 시 replacement pool에 overdue/동반출현 신호 반영)
- **D-2/5/6**는 장기 과제로 두고, 단기에는 D-1/3/4 위주로 반복 개선

---

### 연구 결과 반영 규칙

1. **변경 전 반드시 Baseline 점수 기록** (현재 speed 배열 + seed 42 기준)
2. **단일 변수만 변경** — 한 번에 하나의 파라미터/로직만 수정하여 효과 측정
3. **3등 이상 당첨이 1회라도 나오면 해당 설정을 즉시 `TWENTY_BASE_SPEEDS`에 반영**
4. **5등만 증가하고 4등 이상이 줄면 해당 변경은 기각** (작업지시 [중요] 원칙 준수)
5. **모든 실험 결과는 `당첨 이력.md`와 함께 커밋** — 재현 가능성 보장

## 효율적인 분석 실행 규칙 (시간·비용 최소화 + 상위등수 우선)

> 목적: **불필요한 풀런(52회차×20세트×다중시드×대규모 그리드)**을 줄이고, 상위 등수(3등 이상) 개선에 집중한다.

### 1) 실행 순서(항상 이 순서로)
1. **Baseline 1회 고정**: seed=42 + `start_strategy=blended`로 52회차 평가를 1회 실행하고 결과를 기준선으로 잠근다.
2. **미니 검증 → 풀 검증**: 새 아이디어/필터/가중치 적용 시, 먼저 **짧은 구간(예: 52회차 중 일부 구간)**에서 빠르게 이상 여부를 본 뒤 통과한 것만 52회차 전체로 확장한다.
3. **E-3 다중 시드**: 전체 52회차에서 상위등수 비악화가 확인된 후보만 다중 시드로 확장 검증한다.

### 2) 그리드/튜닝 효율 규칙(거친→정밀)
- `--refine-set` 그리드는 **3단계로 좁힌다**:
  - 1차(거친 탐색): `--refine-step 0.05`
  - 2차(중간 탐색): `--refine-step 0.02`
  - 3차(정밀 탐색): `--refine-step 0.01` (필요할 때만)
- 미당첨 세트처럼 대상이 여러 개면 **병렬 실행 금지**(CPU 과부하로 총시간이 늘 수 있음). **세트별 순차 실행**으로 진행한다.

### 3) 다중 시드(E-3) 비용 절감 규칙
- 기본은 seed 1~20으로 1차 안정성만 확인하고,
- 1차에서 **상위 등수(1~4 또는 1~3) 지표가 흔들릴 때만** seed 1~100으로 확대한다.
- **최종 채택(코드/속도 배열 반영) 직전에는 반드시 seed 1~100 검증을 통과**한다.
- (주의) seed 영향이 거의 없는 변경이라면 1차 seed 1~20 단계에서 중복 반복을 피한다.

### 4) 채택/기각 기준(자동 롤백 포함)
- 어떤 변경이든 **4등 이상(또는 3등 이상)**이 baseline 대비 감소하면 즉시 **기각 및 롤백**한다.
- 5등 증가는 참고 지표로만 사용하고, 상위 등수 악화를 정당화하지 않는다.

### 5) 결과 기록 최소 단위
- “무엇을 바꿨는지(단일 변수)” + “Baseline 대비 상위 등수 변화” + “재현 커맨드” 3가지만 남기고,
- 나머지 과도한 로그/조합 리스트는 남기지 않는다(문서·실행 부담만 증가).
