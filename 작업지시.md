## 방법 3. 카이제곱 검정 (Chi-square Goodness-of-Fit)

### 개요
각 번호의 **이론적 기대 출현 횟수**와 **실제 출현 횟수** 사이의 차이를 통계적으로 검정하여, 기대값보다 유의미하게 적게 나온 번호를 저빈도로 판별한다.

### 계산 방식
1. 총 추첨 회차 수 = `N`, 번호 하나당 기대 출현 횟수 = `E = N × 7 / 45` (num1~num6 + bonus_num 7개 모두 집계 시).
2. 각 번호의 실제 출현 횟수 `O(n)`.
3. 카이제곱 기여값: `χ²(n) = (O(n) - E)² / E`.
4. `O(n) < E` 이면서 `χ²(n)` 이 임계값 이상인 번호를 유의미한 저빈도로 판정.

### 장점
- 통계적 근거가 가장 탄탄하다.
- 단순히 "적게 나왔다"가 아니라 **우연이 아닐 가능성**까지 고려한다.
- 표본(회차 수)이 많을수록 검정력이 높아진다.

### 단점
- 로또는 매 시행이 독립적이므로 **카이제곱 검정 자체의 전제**(특정 번호가 편향될 물리적 이유)가 성립하지 않는다.
- 계산 결과를 사용자에게 직관적으로 설명하기 어렵다.
- 구현 복잡도가 가장 높다.

### 현재 구조 호환성
- **중간** — 백엔드에 새로운 집계 API 추가 필요 (기존 윈도우 API와 별도).
- 프론트엔드에서 계산하려면 카이제곱 라이브러리 또는 직접 구현 필요.

### 도입 검토 메모
- 통계적 타당성은 높지만 로또의 독립 시행 특성상 실용적 의미는 제한적.
- 연구·분석 탭에서 **참고 지표**로 제공하는 것은 의미 있으나, 번호 생성 로직 핵심에 넣기에는 과잉일 수 있다.
