# 03. 실제 개발 진행 룰

이 단계는 `02-guide`에서 작성된 아주 구체적인 **Step-by-Step 개발 가이드**를 읽고, 그에 맞추어 실제 코드(.ts, .tsx, .py) 구현을 진행하는 단계입니다.

## 🎯 사용 스킬

| 역할 | 스킬명 | 용도 |
|------|--------|------|
| 프론트 구현 | `fullstack-developer` | React/Next.js 컴포넌트 및 클라이언트 연동 개발 |
| 백엔드 구현 | `python-expert` | FastAPI 엔드포인트, DB 스키마 및 비즈니스 로직 작성 |
| 데이터/DB | `data-analyst` | SQL 최적화 및 pandas 데이터 파이프라인 개발 |
| 리팩토링 | `component-refactoring` | 비대해진 코드를 적절히 모듈 분리 |
| 테스트 | `frontend-testing` | Vitest + RTL 단위/통합 테스트 작성 |
| 코드 포맷팅 | `fix` | Lint 오류 및 포맷팅 자동 수정 |

## 🚨 제미나이 3 Flash 개발 주의사항
- **환각 방지**: 반드시 주어진 Step의 세부 지시사항 내에서만 코딩하세요. 문서에 없는 코드는 함부로 추측하여 작성하지 않습니다.
- **차근차근 개발**: Step 1을 완전히 짰는지 확인하고, 이상이 없으면 다음 Step 2로 진행합니다. 모든 것을 한 번에 하려다 에러를 쏟아내지 마세요.

## 📝 워크플로우 순서

1. **가이드 숙지**
   - 작업할 현재 Step 명세(수정 파일 위치, 필요한 로직 등)를 정확히 확인합니다.

2. **코드 구현 (Execute)**
   - 지시된 파일 경로에 맞춰 개발을 진행합니다.
   - 프로젝트의 클린 아키텍처 규칙, 프레임워크 제약(Next.js App Router, FastAPI 등)을 준수합니다.

3. **자체 검증 및 에러 체킹**
   - 작성한 코드에 기본적인 빌드/린트 에러가 없는지 체크하고, 코드가 명세서에 부합하는지 리뷰합니다. 

4. **다음 Step 준비**
   - 현재 Step이 성공적으로 마무리되었음을 표시하고 다음 Step 코드 작성을 이어갑니다.
