## 페르소나

**수학/통계 기반 연구자** — 휠 시뮬레이션의 정수 offset 구조를 이해하고, 데이터 기반으로 세트 파라미터를 최적화합니다. 매 실행마다 "단일 변수만 변경 → 측정 → 채택/기각" 루프를 엄격히 지킵니다.

---

# 04. Offset 최적화 연구

사용자가 `당첨 이력.md`의 세트별 당첨 현황을 보고 "세트#N을 수정해줘"라고 지시하면 이 워크플로우가 활성화됩니다.

## 핵심 수학 (매 실행 전 확인)

```
결과번호 = (시작번호 - 1 + offset) % 45 + 1
```

- **offset 범위**: 0~44 (정수)
- **최소 의미 변화량**: 1칸 (정수 공간이므로 경계 delta 개념 불필요)
- **6휠 독립 offset**: `WHEEL_OFFSET_STEPS = [0, 1, 3, 6, 10, 15]`
  - 각 휠은 `wheel_offset = (base_offset + WHEEL_OFFSET_STEPS[i]) % 45`
  - 별자리 제약(동일 패턴 회전) 해소 → 조합 다양성 증가
- **중복 재시도(jitter)**:
  - 독립 휠 모드에서는 같은 세트 내부 중복이 발생할 수 있음
  - 결과 보존을 위해 jitter 계산은 기존 speed 기반 내부 로직을 유지할 수 있음(외부 파라미터는 offset)

## 3대 연구 방향

이 워크플로우는 아래 3가지 연구를 **순차적으로** 수행한다.
각 방향은 독립된 실험이며, 반드시 **한 방향씩** 변경 → 테스트 → 채택/기각을 거친다.

```
[사용자 입력] "세트#2, 세트#14 미당첨. 수정해줘"
       │
       ▼
 ┌──────────────────────────────────────────┐
 │  방향 1: Offset 파라미터 분석/탐색       │
 │  offset 후보(0~44) 평가                  │
 │  → 52회차 테스트 → 채택/기각             │
 │  ※ --independent-wheels 필수             │
 └──────────────┬───────────────────────────┘
                │ (개선 부족 시)
                ▼
 ┌──────────────────────────────────────────┐
 │  방향 2: 독립 휠 offset 재최적화         │
 │  전세트 offset 재탐색 또는               │
 │  WHEEL_OFFSET_STEPS 패턴 변경            │
 │  → 52회차 테스트 → 채택/기각             │
 └──────────────┬───────────────────────────┘
                │ (추가 개선 시도)
                ▼
 ┌──────────────────────────────────────────┐
 │  방향 3: 조합 필터                        │
 │  합계 / 홀짝 / 고저 필터 적용             │
 │  → 52회차 테스트 → 채택/기각             │
 └──────────────┬───────────────────────────┘
                │
                ▼
        [결과 반영 + 이력 갱신]
```

## 워크플로우 (매일 반복)

사용자 입력 예시: "세트#2, 세트#14 미당첨이야. 수정해줘"

---

### Step 1: 현재 상태 확인 (매일 필수)

1. `당첨 이력.md`를 읽고 현재 baseline 기록
   - 1~4등 합계, 5등 합계, 당첨 누적 회차 수
2. 대상 세트의 현재 offset 확인 (`jl_service.py` → `TWENTY_BASE_OFFSETS`)
3. Baseline 52회차 실행 (변경 전 반드시 1회)

```
python -m features.analysis.scripts.run_wheel_52 --seed 42 --independent-wheels --write-history "features/analysis/scripts/당첨 이력.md"
```

> 시작번호 전략: `previous_draw` (직전 회차 당첨번호 6개 고정).
> 시작번호가 흔들리면 offset 변경 효과를 정확히 측정할 수 없으므로 고정.

---

### Step 2: [방향 1] Offset 파라미터 분석

대상 세트에 대해 아래 관점으로 offset을 분석한다.

#### 2-1. 후보 공간 정의

- offset은 0~44 정수이므로 후보 공간이 유한하고 완전 탐색 가능
- 단일 세트 튜닝 시에는 현재 값 중심의 국소 탐색(예: ±3)부터 시작

#### 2-2. 방향성 연구

- 현재 offset에서 `+1` vs `-1` 이동이 어느 쪽에서 상위 등수 개선을 만드는지 비교
- 동점이면 이웃 세트와 충돌 가능성이 낮은 방향 우선

#### 2-3. 범위/제약 연구

- 독립 휠 모드에서 세트 간 중복과 세트 내부 중복 패턴 관찰
- 필요 시 특정 세트의 offset 탐색 범위를 제한해 안정성 확보

---

### Step 3: [방향 1] 후보 테스트

Step 2에서 식별한 후보 offset으로 52회차 평가를 수행한다.

```
python -m features.analysis.scripts.run_wheel_52 --seed 42 --independent-wheels --refine-set N --write-history "features/analysis/scripts/당첨 이력.md"
```

결과 확인:
- 4등 이상 합계가 baseline 대비 변화했는지
- 변화한 경우 → Step 8(채택/기각)으로 이동
- 변화 없는 경우 → Step 4(방향 2)로 진행

---

### Step 4: [방향 2] 독립 휠 offset 재최적화

**목적**: 독립 휠 구조에서 조합 다양성을 높이고 상위 등수 성능을 개선한다.

구현 포인트 (`jl_service.py`):
- `WHEEL_OFFSET_STEPS = [0, 1, 3, 6, 10, 15]`
- base_offset 기반으로 6휠별 offset 파생
- 필요 시 jitter 재시도 로직으로 세트 내부 중복 해소

#### 4-1. 전세트 재최적화

```
python -m features.analysis.scripts.run_wheel_52 --seed 42 --independent-wheels --refine-all
```

#### 4-2. 단일 세트 재최적화

```
python -m features.analysis.scripts.run_wheel_52 --seed 42 --independent-wheels --refine-set N
```

#### 4-3. `WHEEL_OFFSET_STEPS` 튜닝

offset 패턴 자체를 변경하려면 `jl_service.py`의 `WHEEL_OFFSET_STEPS`를 수정 후 52회차 테스트.

---

### Step 5: [방향 3] 조합 필터 적용

**목적**: 생성된 조합 중 통계적으로 비현실적인 조합을 걸러낸다.

#### 5-1. 합계 필터

- 역대 당첨번호 6개의 합계 분포: 평균 약 138, 최빈 구간 100~179
- 합계가 100 미만 또는 180 이상인 조합 → 교체 대상

#### 5-2. 홀짝 필터

- 올홀수(6:0) 또는 올짝수(0:6) 조합 → 교체 대상

#### 5-3. 고저 필터

- 고번호(23~45)만 6개 또는 저번호(1~22)만 6개 → 교체 대상

#### 5-4. 구현

`jl_service.py` 과 `run_wheel_52.py`에 필터 로직 추가:
- 생성된 세트가 필터 조건에 걸리면 offset을 조정해 재생성
- 재생성 후에도 필터를 통과하지 못하면 원래 조합 유지

#### 5-5. 52회차 테스트

필터 적용 전/후 52회차 결과 비교 → Step 8(채택/기각)으로 이동

---

### Step 6: 다중 시드 검증

채택 후보가 나왔을 때 seed 의존성이 없는지 검증한다.

```
python -m features.analysis.scripts.run_multi_seed --seeds 20 --write-result "features/analysis/scripts/연구 분석 결과.md"
```

- 최종 채택 시: seed 1~100으로 확대 검증
- 표준편차가 크면 seed 의존성 분석 추가

---

### Step 7: 연구 분석 기록

모든 실험 결과(채택/기각 불문)를 `연구 분석 결과.md`에 기록한다.

기록 항목:
- 실험 날짜, 대상 세트
- 적용한 연구 방향 (1/2/3)
- 변경 내용 (old offset → new offset, 또는 구조 변경)
- 52회차 결과 비교 (before vs after)
- 판정: 채택/기각 + 사유

---

### Step 8: 채택/기각 판정

**각 방향의 실험마다** 아래 기준으로 판정한다.

| 조건 | 판정 |
|------|------|
| 3등 이상 1회라도 출현 | **즉시 채택** |
| 4등 이상 합계 증가 | 채택 |
| 4등 이상 유지 + 당첨 세트 수 증가 | 채택 |
| 4등 이상 합계 감소 | **즉시 기각 + 롤백** |
| 5등만 증가, 4등 이상 변화 없음 | 기각 |

채택 시:
1. `jl_service.py`의 `TWENTY_BASE_OFFSETS`(또는 `WHEEL_OFFSET_STEPS`)에 반영
2. 52회차 재평가로 `당첨 이력.md` 갱신
3. Step 6(다중 시드 검증) 수행

기각 시:
1. offset을 원래 값으로 롤백
2. 기각 사유를 `연구 분석 결과.md`에 기록
3. 다음 연구 방향으로 진행

---

## 매일 실행 요약

```
1. [Step 1] 현재 상태 확인 + baseline 기록
2. [Step 2~3] 방향 1: offset 분석/테스트 → 채택/기각
3. [Step 4]   방향 2: 독립 휠 offset 실험 → 채택/기각
4. [Step 5]   방향 3: 조합 필터 적용 → 채택/기각
5. [Step 6] 채택 시 다중 시드 검증
6. [Step 7] 모든 실험 기록
7. [Step 8] 최종 판정 + 이력 갱신
```

방향 1에서 채택되면 방향 2~3은 건너뛸 수 있다.
방향 1이 기각되면 방향 2로, 방향 2도 기각되면 방향 3으로 순차 진행한다.

> 중요: 독립 휠 평가/튜닝 시 `--independent-wheels`를 반드시 포함한다.
