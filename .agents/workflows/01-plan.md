# 01. 플랜 (기획 및 수정 위치 지정)

이 단계는 실제 구현을 시작하기 전에 프로젝트의 목표를 정의하고, 기능 구현을 위한 전체적인 플랜과 수정할 파일의 위치를 명확히 지시하는 기획 단계입니다. 기존의 상세 설계(디테일 플랜)가 이 단계로 통합되었습니다.

## 🎯 사용 스킬

| 역할 | 스킬명 | 용도 |
|------|--------|------|
| 핵심 | `project-planner` | 태스크 분할, 마일스톤, 의존성 파악 |
| 전략 | `strategy-advisor` | 기술적 대안 방향 설정 |
| UI 설계 | `tailwind-component-architect` | Tailwind 기반 컴포넌트 정의 및 설계 |
| API 설계 | `orpc-contract-first` | 프론트-백엔드 간 타입 안전한 API 규격 정의 |
| 백엔드 | `python-expert` | Pydantic 모델, DB 스키마 구상 |
| 성능 고려 | `vercel-react-best-practices` | 서버/클라이언트 컴포넌트 렌더링 검토 |

## 🎯 목적
- 사용자의 요구사항을 분석하여 명확한 목표를 세웁니다.
- 개발을 위한 로직과 수정될 파일의 경로를 확정하여 다음 설계 가이드 단계로 넘겨줍니다.

## 📝 워크플로우 순서

1. **목표 정의 (Define Success)**
   - 사용자의 의도를 분석하고, 무엇을 만들어야 하는지 명확히 정리합니다.

2. **기술적 플랜 수립 (High-level Plan)**
   - 기능 구현에 필요한 API 명세, DB 구조의 변경점, 프론트엔드 UI/상태 관리 등 핵심 기술 전략을 구상합니다.

3. **정확한 수정 위치 지정 (Target Locations)**
   - `frontend/src/` 및 `backend/` 하위의 구체적인 파일 경로 리스트를 뽑아, 어느 파일을 수정하거나 추가할지 목록화합니다.

4. **플랜 문서 출력 (Output)**
   - 분석한 플랜 내용과 수정 대상 파일의 정확한 위치/경로를 지시사항 형태로 간략히 정리하여 출력합니다.
   - 단, 코드는 직접 작성하지 않습니다.
