# 04. 개발 (디테일 플랜의 코드 구현 및 자동 수정)

이 단계는 `03-verify-plan`을 통해 무결성이 입증된 `implementation-plan.md` 문서의 제안을 **실제 프로젝트 코드(.ts, .tsx, .py)로 변환하고 구현**하는 최종 단계입니다. 오로지 설계서에 기반하여 흔들림 없이 코딩합니다.

## 🎯 사용 스킬 지침
이 단계에서는 실제 코드를 생성하고 오류를 잡는 파워풀한 스킬들을 사용합니다:

1. **`frontend-testing`**:
   - 개발과 동시에(혹은 직후) 구현된 주요 핵심 로직과 UI 컴포넌트에 대해 Vitest 및 React Testing Library 스펙 파일을 작성합니다.
2. **`python-expert` & `debugger`**:
   - 백엔드 FastAPI 구현 도중 발생하는 버그나 타입 에러를 추적하고, Pythonic한 코드로 깔끔하게 디버깅합니다.
3. **`fix`**:
   - 코드 작성 완료 후 터미널 창(또는 편집기)에서 발생하는 린트(Lint) 에러나 자잘한 오타, 포맷팅 이슈가 발생하면 즉각 동작하여 코드를 자동 수정하고 빌드(CI)를 통과하게 만듭니다.
4. **`component-refactoring`**:
   - 코드를 구현하다가 특정 컴포넌트 범위가 의도치 않게 거대해질 경우, Hooks나 분리된 하위 컴포넌트로 코드를 리팩토링합니다.

## 📝 워크플로우 진행 순서

1. **설계 기반 스캐폴딩(Scaffolding)**
   - `implementation-plan.md`에 나열된 순서대로 새 파일들을 생성하고, 인터페이스(타입/기본 함수뼈대)를 먼저 정의합니다.
2. **비즈니스 로직 및 UI 구현**
   - Tailwind CSS 적용 규칙과 FastAPI 라우팅 규칙에 맞게 디테일 로직을 작성합니다. 절대로 설계서와 다른 방향으로 개발하지 않습니다.
3. **코드 레벨 디버깅 및 Fix**
   - `run_command` 도구를 사용해 빌드나 테스트를 백그라운드에서 실행하고, 에러 로그 발견 시 `fix`나 `debugger` 스킬의 가이드대로 소스 코드를 교정합니다.
4. **최종 결과 피드백**
   - 개발이 완료되면 사용자가 변경 사항을 쉽게 확인할 수 있도록 `README.md`를 업데이트하거나, 주요 변경 내역 요약을 채팅 창에 제공하고 작업을 마무리합니다.
