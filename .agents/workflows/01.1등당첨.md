# 워크플로: 1등 당첨 offset 선정 (01.1등당첨)

## 입력 형식

- **`P`**: 직전 회차 (`--prev-draw`)
- **`C`**: 1등을 맞출 대상 회차 (`--current-draw`)
- **`세트N`**: 적용 세트 번호 1~20 (`--set-index N`)

`lotto_winners`에 `P`·`C` 행이 있어야 한다.
시작번호는 **직전 회차 1등 번호를 DB 컬럼 순서(num1→num6) 그대로** 사용한다.

## Step A — 0번: 누적 당첨번호 집계

프로젝트 루트에서:

```powershell
python -m features.analysis.scripts.run_00_cumulative_winners `
  --up-to-draw 1215 `
  --top-n 5 `
  --bottom-n 5
```

- `1 ~ up_to_draw` 본번호 누적 빈도를 계산한다.
- 상위 5개(`top_numbers`), 하위 5개(`bottom_numbers`)를 출력한다.

## Step B — 1번: 6개 offset 계산 → pick JSON

프로젝트 루트에서:

```powershell
python -m features.analysis.scripts.run_01_pick_offset_start `
  --prev-draw 1214 `
  --current-draw 1215 `
  --set-index 4 `
  --write-pick pick.json
```

- `P`와 `C`의 1등 번호를 같은 위치끼리 비교해 6개 칸수(`offset_steps`)를 계산한다.
- 추천번호는 `C` 1등 번호에 `offset_steps`를 다시 적용해 만든다(45 초과 시 순환).
- 추천번호에 누적 상위 번호가 포함되면 누적 하위 번호로 교체한다.
- 황금비(`WHEEL_OFFSET_STEPS`)와 시작번호 조합(`start_permutation`)은 사용하지 않는다.

### pick JSON 필드

| 필드 | 설명 |
|------|------|
| `이전회차` | P |
| `현재회차` | C (1등기준회차) |
| `세트번호` | S |
| `오프셋칸수` | 6개 칸수 목록 |
| `이전회차당첨번호_db순서` | P회차 1등 번호(DB 순서) |
| `현재회차당첨번호_db순서` | C회차 1등 번호(DB 순서) |
| `추천번호` | 치환 적용 후 추천 번호 6개 |
| `치환내역` | 상위→하위 번호 치환 상세 |
| `선정시각_UTC` | ISO 시각 |

## Step C — 2번: 단일 세트 최신 52회차 → 당첨 이력.md

```powershell
python -m features.analysis.scripts.run_02_three_year_single_set `
  --pick-json pick.json `
  --write-history "features/analysis/scripts/당첨 이력.md"
```

- 기본 **최신 52회차**.
- pick 없이 실행하려면 `--set-index`, `--offset-steps` 6정수, `--current-draw`(필수)를 직접 전달한다.
- 각 목표 회차 `T`마다 누적 집계는 반드시 `1 ~ (T-1)`로 다시 계산한다.

### 당첨 이력.md

- 표 4열: `세트#` | `offset` | `회차(등수)` | `1등기준회차` — 마지막 열은 **항상 pick의 `current_draw`(C)**.
- 같은 파일을 다시 쓸 때 **다른 세트# 행은 유지**하고, 이번에 평가한 세트 행만 갱신한다. ``### 전체 누적`` 불릿(서로 다른 회차 수·1~5등 건수)은 **병합된 표 전체**를 파싱해 매번 다시 계산한다.

## §7 교차 검증 체크리스트

1. 위 `python -m` 경로가 실제 모듈명과 일치하는가.
2. `P,C,세트N` → CLI 인자 매핑이 위와 같은가.
3. `pick.json` 필드(`offset_steps` 포함)가 2번 로더와 문서 표가 동일한가.
4. `당첨 이력.md` 4열·`1등기준회차=C`가 실제 출력과 일치하는가. `### 전체 누적` 불릿은 표의 `회차(N등)` 합산과 일치하는가(2번 저장 시 자동 재계산).
5. 문서에 황금비/시작번호 조합 관련 구버전 설명이 남아 있지 않은가.
