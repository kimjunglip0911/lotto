# 워크플로: 1등 당첨 offset 선정 (01.1등당첨)

## 입력 형식

- **`P`**: 직전 회차 (`--prev-draw`)
- **`C`**: 1등을 맞출 대상 회차 (`--current-draw`)
- **`세트N`**: 적용 세트 번호 1~20 (`--set-index N`)

`lotto_winners`에 `P`·`C` 행이 있어야 한다. `C != P+1`이면 엔진 기본 규칙과 어긋날 수 있어 경고만 출력하고 지정 `(P,C)`를 그대로 사용한다.

## Step A — 1번: offset × P(7,6)=5040 탐색 → pick JSON

프로젝트 루트에서:

```powershell
python -m features.analysis.scripts.run_01_pick_offset_start `
  --prev-draw 1217 `
  --current-draw 1218 `
  --set-index 1 `
  --write-pick pick.json
```

선택: 전체 후보 누락·1등 존재 여부를 파일에서 검증하려면 `--write-audit-md run_01_audit.md` 를 추가한다(지정 시 1등이 나와도 **226,800행 전부** 순회·기록). **Windows**에서는 같은 명령만으로 **새 PowerShell 창**이 열려 감사 MD가 실시간으로 갱신된다(`Get-Content -Wait`). 창이 필요 없으면 `--no-audit-follow-window` 를 붙인다.

- `P`회차 **본6(정렬) + 보너스** 7개를 시작 풀로 두고, 6휠에는 그중 서로 다른 6개를 배치한다.
- 세트 `S` 한 줄에 대해 `offset` 0~44 × `itertools.permutations(range(7), 6)` (**5040**) 고정 순서. 인덱스 0~5=본6 오름차순, 6=보너스.
- 전체 **45 × 5040 = 226,800** 후보.
- `C`에서 1등이면 즉시 중단 후 pick 기록.
- 없으면 **같은 프로세스 안에서** `C` 단일 회차 보조 점수(가중치) 최대 후보를 확정한 뒤 pick 기록.
- **폴백 타이브레이크**: 가중치·매치 동일 시 **offset 오름차순**, 그다음 **순열 튜플 사전순**.

### pick JSON 필드

| 필드 | 설명 |
|------|------|
| `prev_draw` | P |
| `current_draw` | C (1등기준회차) |
| `set_index` | S |
| `offset` | 선정 offset (0~44) |
| `start_permutation` | 길이 6, 서로 다른 0~6 (본6 정렬→인덱스 0~5, 보너스→6) |
| `prev_bonus` | P회차 보너스 번호 (메타) |
| `start_pick_mode` | `seven_pool_p7_6` (7풀에서 6휠 배치) |
| `rank1_achieved` | `C`에서 1등으로 조기 종료 여부 |
| `fallback_used` | 1등 없이 폴백으로 확정했는지 |
| `picked_at_utc` | ISO 시각 |

## Step B — 2번: 단일 세트 3년치 → 당첨 이력.md

```powershell
python -m features.analysis.scripts.run_02_three_year_single_set `
  --pick-json pick.json `
  --write-history "features/analysis/scripts/당첨 이력.md"
```

- 기본 **최신 156회차** (`THREE_YEAR_DRAW_COUNT`). DB가 부족하면 있는 만큼만 쓰고 stderr에 경고.
- pick 없이 실행하려면 `--set-index`, `--offset`, `--start-permutation` 6정수, `--current-draw`(필수), 선택 `--prev-draw` 로 동일 정보를 직접 전달.

### 당첨 이력.md

- 요약: `당첨 누적 회차`, `1등`~`5등` 건수 (해당 세트 히트만).
- 표 4열: `세트#` | `offset` | `회차(등수)` | `1등기준회차` — 마지막 열은 **항상 pick의 `current_draw`(C)**.

## §7 교차 검증 체크리스트

1. 위 `python -m` 경로가 실제 모듈명과 일치하는가.
2. `P,C,세트N` → CLI 인자 매핑이 위와 같은가.
3. `pick.json` 필드가 2번 로더와 문서 표가 동일한가.
4. `당첨 이력.md` 4열·`1등기준회차=C`가 실제 출력과 일치하는가.
5. `features/analysis/README.md`, `로직.md`, `작업지시.md`, `config.py` 주석에 삭제된 스크립트명이 남아 있지 않은가.
