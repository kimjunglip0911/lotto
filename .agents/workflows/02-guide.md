# 02. 검증 및 개발 가이드

`01-plan` 단계에서 도출된 기획과 파일 수정 지시사항을 바탕으로 사전 에러 검증을 수행하고, 실제 코드를 작성할 AI가 차근차근 개발할 수 있도록 **Step-by-Step (Step 1, Step 2...) 형태의 개발 가이드**를 제공하는 단계입니다.

## 🎯 사용 스킬

| 역할 | 스킬명 | 용도 |
|------|--------|------|
| 기술/보안 리뷰 | `code-reviewer` | 설계 무결성, 아키텍처 제약사항, 오류 사전 검증 |
| 프론트 리뷰 | `frontend-code-review` | 비즈니스 로직, 엣지 케이스 체크 확인 |
| 성능 검증 | `vercel-react-best-practices` | RSC 경계 및 Re-render 패턴 위반 검사 |
| 의사결정 | `decision-helper` | 기술적 분기에 대한 의사결정 시 선택 |

## 🎯 목적
제미나이 3 Flash 모델 등 성능이 낮고 환각(Hallucination) 현상이 발생할 수 있는 모델이 오류 없이 지시사항을 따를 수 있도록, 작업을 잘게 쪼개어 가장 명확하고 정확한 가이드를 제공해야 합니다.

## 📝 워크플로우 순서

1. **설계 사전 검증 (Verify Plan)**
   - 01단계의 기획안 내용 중 아키텍처 규칙 위반, 예외 처리 누락, 의존성 충돌 등이 없는지 엄격히 리뷰합니다.

2. **Step-by-Step 가이드 작성 (Step Breakdown)**
   - 전체 개발 과정을 매우 작고 순차적인 Step으로 분할합니다.
   - 예시 구조:
     - `[Step 1] DB 스키마 추가 (수정 파일: backend/models/xxx.py)`
     - `[Step 2] API 라우터 기본 세팅 (수정 파일: backend/routers/xxx.py)`
     - `[Step 3] 프론트엔드 UI 컴포넌트 뼈대 생성`
     - `[Step 4] API 통신 연동 및 상태 처리`

3. **각 Step별 세부 지시**
   - 각 Step마다 모델이 환각 없이 코딩할 수 있도록 의도, 제약 사항, 구현 방향, 정확한 파일 위치를 적시해야 합니다.

4. **가이드 문서화 (Output)**
   - `implementation-plan.md` 에 Step 1부터 순차적으로 코딩을 따라할 수 있는 명세서를 확정 지어 출력합니다.
