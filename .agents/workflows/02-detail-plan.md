# 02. 디테일 플랜 (기술 상세 설계)

이 단계는 `01-plan`에서 도출된 `implementation-plan.md`를 바탕으로, 실제 작성해야 할 **파일 경로 수립, API 명세, 데이터베이스 스키마 설계, UI 컴포넌트 트리 작성** 등을 수행하는 구체화 단계입니다. 개발 직전의 가장 중요한 청사진을 생성합니다.

## 🎯 사용 스킬 지침
이 단계에서는 다음 스킬들을 종합적으로 구동하여 빈틈없는 설계를 완성하세요:

1. **`tailwind-component-architect`**:
   - 프론트엔드 UI/UX 설계 시, 필요한 React 요소들을 Tailwind 기반의 독립적이고 재사용 가능한 컴포넌트(Component Tree)로 분리하고 명세합니다.
2. **`orpc-contract-first`**:
   - 프론트엔드와 백엔드가 데이터를 주고받을 API 엔드포인트(URL, 파라미터, 응답 타입)를 Contract-First 기반의 타입 안전한 명세서로 작성합니다.
3. **`python-expert` & 데이터베이스 스킬**:
   - 백엔드(FastAPI) 개발 시 필요한 Pydantic 모델과 SQLite/DB 스키마, 주요 로직의 흐름을 설계합니다.
4. **`vercel-react-best-practices`**:
   - 컴포넌트 설계 시 불필요한 렌더링을 막고, 서버 컴포넌트(RSC)와 클라이언트 컴포넌트를 명확히 구분하도록 설계에 강제합니다.

## 📝 워크플로우 진행 순서

1. **초안 문서 분석**
   - 기존에 작성된 `implementation-plan.md`의 목표와 마일스톤을 깊이 있게 읽고 기술적 맥락을 파악합니다.
2. **디렉토리 및 파일 구조 설계**
   - `frontend/src/` 및 `backend/` 하위에 새로 생성하거나 수정해야 할 정확한 파일 절대/상대 경로 리스트를 작성합니다.
3. **API & 데이터베이스 명세**
   - 프론트-백엔드 간 통신 규격(Request/Response Type)과 백엔드 DB 테이블 스키마를 마크다운 표 형식으로 꼼꼼히 설계합니다.
4. **컴포넌트 설계 (Props & State)**
   - 프론트엔드 UI 컴포넌트들의 트리 구조를 그리고, 각 컴포넌트가 받을 Props와 내부 상태(State) 로직을 구상합니다.
5. **디테일 플랜 업데이트**
   - 설계된 모든 내용을 `implementation-plan.md` 문서의 **[상세 기술 설계]** 챕터에 병합(또는 덮어쓰기)하여 코딩을 바로 시작할 수 있는 수준의 문서를 완성합니다.
