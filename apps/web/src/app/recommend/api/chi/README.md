# chi API 모듈

## 목적
- chi 추천 화면에서 사용하는 당첨 번호 이력 조회 로직을 제공한다.
- 조회 책임을 `fetchDraws.ts`, `fetchRange.ts`, `chiHistory.ts`로 나눠 유지보수를 쉽게 만든다.

## 실행 방법
- 상위 코드에서 `fetchChiSquareFullHistory(apiUrl)`를 호출한다.
- 내부 동작 순서
  - `fetchDraws.ts`: 회차 목록 조회 후 숫자 회차만 정제
  - `chiHistory.ts`: 빈 회차 여부 판단 및 최신 회차 계산
  - `fetchRange.ts`: `latest + 1` 상한 규칙으로 범위 조회 후 행 검증

## 환경 변수
- 이 모듈은 직접 환경 변수를 읽지 않는다.
- 호출부에서 전달한 `apiUrl`을 사용해 서버 주소를 결정한다.

## 주의사항
- 회차 목록 응답이 배열이 아니면 즉시 예외를 던진다.
- 범위 조회 응답이 배열이 아니면 즉시 예외를 던진다.
- 회차 목록이 비어 있으면 범위 조회를 하지 않고 `[]`를 반환한다.
- 범위 조회는 `draw_no` 상한이 미포함이므로 `latest + 1`을 사용해야 최신 회차가 포함된다.
