# JL 메인 서비스 — 현재 로직 요약

## 개요

현재 메인 서비스(`features/analysis/api/jl_service/generator.py`)는
**1번 스크립트 결과(`pick.json`)를 기준으로 20세트 전체를 동일 규칙으로 생성**한다.

- 황금비(`WHEEL_OFFSET_STEPS`) 기반 생성 로직은 메인 생성 경로에서 사용하지 않는다.
- 시작번호 조합(순열) 로직은 메인 생성 경로에서 사용하지 않는다.
- 시작번호는 직전 회차 1등 번호를 **DB 컬럼 순서(`num1→num6`) 그대로** 사용한다.

> 로또는 난수 추첨이므로 당첨 보장 불가. 참고·실험용.

---

## 핵심 규칙

### 1) 시작번호
- 기준 회차 `draw_no` 생성 시, `draw_no-1` 회차의 `num1~num6`를 그대로 사용한다.

### 2) 이동 공식
각 위치 `i(0~5)`에 대해:

```
result[i] = (start[i] - 1 + offset_steps[i]) mod 45 + 1
```

- `offset_steps`는 `pick.json`의 6개 칸수를 기준으로 하되,
  세트별 `TWENTY_BASE_OFFSETS[set]`를 합성해 최종 칸수를 만든다.

### 3) 누적 상/하위 번호 치환
- 누적 집계 범위: `1 ~ (draw_no-1)`
- 누적 상위 5개 번호(`hot`)가 결과에 포함되면
  누적 하위 5개 번호(`cold`)로 치환한다.
- 치환 후 중복 번호가 생기면 `cold` 우선으로 중복 해소한다.

---

## 메인 서비스 동작 순서

```
1. 프로젝트 루트 pick.json 로드
   - 유효 키: set_index, offset_steps(6개)

2. 20세트 루프
   - 기본 offset_steps는 공통 사용
   - 세트별 offset(`TWENTY_BASE_OFFSETS`)을 합성해 세트마다 다른 최종 칸수 적용
   - 시작번호는 draw_no-1의 num1~num6 순서 고정
   - 누적 상/하위 번호 치환 적용

3. 결과 20세트 반환/저장
```

설명:
- 현재 구현은 `pick.json`이 존재하면 20세트 모두 같은 규칙을 따르되,
  세트별로 최종 칸수가 달라져 서로 다른 추천번호가 생성된다.

## 기존 로직과의 관계

아래 로직들은 코드에 남아 있지만, **pick.json 연동 메인 생성 경로에서는 우선 사용되지 않는다**.

- `TWENTY_BASE_OFFSETS`
- `WHEEL_OFFSET_STEPS`
- 시작번호 순열 다양화(`start_permutation_overrides`, `START_PERMUTATIONS`)
- 기존 `physics.py` 기반 6휠 이동/지터 재시도 경로

즉, 현재 메인 생성의 실질 기준은:
- `pick.json`의 `offset_steps`
- 직전회차 번호 순서 고정
- 누적 상/하위 치환
