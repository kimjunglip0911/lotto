# JL 휠 — 로직 아키텍처

## 개요

직전 회차 당첨번호를 시작점으로, 45칸 원형 그리드 위에서 세트별 **offset**(이동 칸수)만큼 이동하여 번호를 결정한다. 20세트를 생성하며, 세트마다 시작번호의 **조합**(배치 순서)과 **offset**을 다르게 적용한다.

> 로또는 난수 추첨이므로 당첨 보장 불가. 참고·실험용.

---

## 핵심 공식

```
결과번호 = (시작번호 - 1 + offset) mod 45 + 1
```

이것이 전부다.

- **시작번호**: 직전 회차 당첨 본번호 6개 (고정, 전략 선택 없음)
- **offset**: 세트별로 config에 정의된 이동 칸수 (0~44)
- **45칸 원형 그리드**: 1→2→...→45→1→... 순환

### 독립 휠 (6개 번호 생성, 항상 적용)

6개 시작번호 각각에 대해 **독립적으로** 이동한다:

```
결과[i] = (시작번호[i] - 1 + 세트_offset + 휠_step[i]) mod 45 + 1
```

- `세트_offset`: `TWENTY_BASE_OFFSETS[세트번호]` (0~44)
- `휠_step[i]`: `WHEEL_OFFSET_STEPS = [0, 7, 17, 24, 34, 41]` (황금각 기반)

#### 황금각(Golden Angle) 분산 원리

황금비 φ = (1+√5)/2 ≈ 1.618에서 파생:

```
황금각 스텝 = 45 × (2 − φ) ≈ 17.189칸
```

각 휠은 이전 휠에서 ~17.19칸씩 이동하여 45칸 원형 그리드 위에 비주기적·균일 분포를 형성한다.
매번 새 점이 기존 가장 큰 빈 구간에 배치되어 클러스터링을 방지한다.

### 세트별 시작번호 조합 다양화

동일한 6개 번호를 사용하되, 세트마다 **순열(배치 순서)**을 다르게 한다.
`config.py`의 `START_PERMUTATIONS`에 20개 순열이 정의되어 있다:

```
#1  (0,1,2,3,4,5)  원본      #11 (3,2,1,0,5,4)  역회전4
#2  (1,2,3,4,5,0)  회전1     #12 (4,3,2,1,0,5)  역회전5
#3  (2,3,4,5,0,1)  회전2     #13 (0,2,4,1,3,5)  교차1
#4  (3,4,5,0,1,2)  회전3     #14 (1,3,5,0,2,4)  교차2
#5  (4,5,0,1,2,3)  회전4     #15 (2,4,0,3,5,1)  교차3
#6  (5,0,1,2,3,4)  회전5     #16 (3,5,1,4,0,2)  교차4
#7  (5,4,3,2,1,0)  역순      #17 (4,0,2,5,1,3)  교차5
#8  (0,5,4,3,2,1)  역회전1   #18 (5,1,3,2,4,0)  교차6
#9  (1,0,5,4,3,2)  역회전2   #19 (0,3,1,5,2,4)  혼합1
#10 (2,1,0,5,4,3)  역회전3   #20 (4,2,5,0,3,1)  혼합2
```

- 인덱스 0~5는 정렬된 시작번호의 위치에 대응
- 20개 모두 고유한 순열 → 세트마다 6개 휠에 다른 번호가 매핑
- **튜닝**: `START_PERMUTATIONS` 리스트를 직접 수정하여 배치 전략 변경 가능

### 중복 번호 처리

6개 휠 결과에 같은 번호가 나오면, 해당 휠의 이동량을 조금씩 늘리거나 줄여서 다른 번호가 나올 때까지 재시도한다.

---

## 조합 교정 (항상 적용)

생성된 6번호가 아래 기준을 위반하면 자동 교체:

| 조건 | 허용 범위 | 교체 방식 |
|------|----------|----------|
| 합계 | 100 ~ 175 | 고빈도 번호 → 저빈도 번호 (크기 조정) |
| 홀짝 | 홀2:짝4 ~ 홀4:짝2 | 고빈도 → 가장 가까운 반대 홀짝 번호 |
| 고저 | 고2:저4 ~ 고4:저2 (기준 23) | 고빈도 → 가장 가까운 반대 고저 번호 |

교정은 2회 적용: (1) 세트 생성 직후, (2) 사후 중복 교체 후.

## 튜닝 파라미터 (`config.py`)

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `TWENTY_BASE_OFFSETS` | `[15, 12, 25, ...]` (20개) | 세트별 기준 이동 칸수 |
| `START_PERMUTATIONS` | 20개 순열 | 세트별 시작번호 배치 순서 |
| `WHEEL_OFFSET_STEPS` | `[0, 7, 17, 24, 34, 41]` | 6개 휠 간 추가 이동량 (황금각 분산) |
| `PER_SET_WHEEL_STEPS` | 세트별 커스텀 휠 step | 특정 세트만 별도 패턴 적용 |
| `_JITTER_BASE_SPEEDS` | 20개 float | 레거시. jitter 재시도 시 기준점 역할 |
| `COMBO_FILTER_SUM_MIN/MAX` | 100 / 175 | 6번호 합계 허용 범위 |
| `COMBO_FILTER_ODD_MIN/MAX` | 2 / 4 | 홀수 개수 허용 범위 |
| `COMBO_FILTER_HIGH_MIN/MAX` | 2 / 4 | 고번호(≥23) 개수 허용 범위 |

## 모듈 구조 (`features/analysis/api/jl_service/`)

```
jl_service/
├── __init__.py        ← 공개 API 재수출 (하위 호환 유지)
├── config.py          ← 상수·오프셋·휠 스텝·조합 교정 기준 (튜닝 대상)
├── physics.py         ← 그리드 이동 + 6휠 드로우
├── start_numbers.py   ← 시작번호 (직전 회차 당첨번호 6개)
├── dedup.py           ← 중복 방지 (사전 다양화 + 사후 빈도 교체)
├── generator.py       ← 20세트 생성 파이프라인 + 조합 교정
├── analysis.py        ← 당첨 비교·중복 인사이트 분석
└── search.py          ← 파라미터 역탐색
```

| 모듈 | 핵심 | 설명 |
|------|------|------|
| **config** | `TWENTY_BASE_OFFSETS` | 세트별 offset. 튜닝 시 이 파일만 수정. |
| **physics** | `draw_six` | 6휠 이동 + 중복 번호 재시도. |
| **start_numbers** | `get_previous_draw_winning_starts` | 직전 회차 당첨 본번호 6개. |
| **dedup** | `_draw_unique_set_with_variants` | 사전: 시작번호 회전. 사후: 고빈도→저빈도 번호 치환. |
| **generator** | `generate_jl_wheel_sets` | 전체 파이프라인 오케스트레이션 + 조합 교정. |
| **analysis** | `analyze_draw_duplicate_sets` | 교체 전/후 비교 + 등수 판정. |
| **search** | `find_offset_for_number` | 특정 번호를 만드는 offset 역탐색. |

## 생성 파이프라인 (20세트)

```
1. 시작번호 결정
   └─ 직전 회차 당첨번호 6개 (무조건)

2. 세트별 루프 (i = 0..19)
   ├─ offset = TWENTY_BASE_OFFSETS[i]
   ├─ 시작번호 조합 다양화 (세트별 회전 + 교환)
   ├─ 6휠 독립 이동 (offset + 황금각 step → 결과번호)
   │   └─ 중복 번호 시 이동량 조정하여 재시도
   ├─ 조합 교정 (합계/홀짝/고저 위반 시 고빈도→저빈도 번호 교체)
   └─ 사전 중복 검사 → 이전 세트와 겹치면 재생성

3. 사후 중복 교체
   └─ 세트#1 유지, #2~#20 중 중복 세트의
      고빈도 번호 1개를 저빈도 풀에서 치환

4. 2차 조합 교정 (사후 교체로 인한 재위반 방지)

5. 결과 반환 (20세트 × 6번호)
```
