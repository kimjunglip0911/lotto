# 누적번호 분석 리포트 (핵심 요약)

## 1) 이번 버전 핵심 (분산 제약 + 동적 Top-N)

- 평가 범위: **100회 ~ 최신(1221회)**, 총 **1122회**
- 기간 후보 수: **128개** (100개 이상)
- 기준 전략: **평균근접(`nearestMean4`) + 상2+하2(`twoHotTwoCold`)**
- 각 전략은 **Top-N 후보풀(8개)**에서 동적으로 2개 기간을 선택
- 최종 4개 선정 시 하드 제약 적용:
  - 번호대 분산(1~15/16~30/31~45 각 구간 최대 2개)
  - 홀짝 쏠림 금지(4개 모두 홀/짝 금지)
  - 연번 과다 방지(연속쌍 1개 이하)
  - 동일 끝수 금지

## 2) 전략별 선택 기간(현재, 동적)

- 평균근접: **136, 28**
- 상2+하2: **336, 392**

## 3) 변경 로직 최종 4개 과거 평가 결과

- 최신 기준 예시 최종 4개: **5, 8, 20, 26**
- `>=1` 적중률: **49.64%**
- 평균 적중: **0.590**
- 최대 연속 미적중: **8회**

## 4) 해석 (판단용)

- 전략 결합(평균근접+상2+하2) 최종 4개 기준 `>=1` 적중률은 **49.64%**
- 이전 대비 적중률은 소폭 하락했지만, **최대 연속 미적중이 10회 → 8회로 개선**
- 즉, 수익률 극대화보다 연속 손실 완화를 우선한 방어형 세팅으로 해석 가능

## 5) 현재 번호 선정 규칙 (간단)

1. 각 전략별로 Top-N(8개) 후보풀을 만들고, 간격 제약(min gap)으로 2개 기간을 동적 선택
2. 기간별 점수를 합산해 전략별 4개 번호 선정(평균 4 + 상하 4)
3. 두 전략 8개에서 공통번호 우선으로 후보 생성
4. 분산 하드 제약(번호대/홀짝/연번/끝수)을 통과하는 조합 중 최고 점수 4개를 최종 확정

cd frontend
npm run accumulated-numbers:strategy-metrics
