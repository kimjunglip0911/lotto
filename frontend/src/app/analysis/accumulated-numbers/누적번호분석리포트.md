# 누적번호 분석 리포트 (핵심 요약)

## 1) 이번 버전 핵심

- 평가 범위: **100회 ~ 최신(1221회)**, 총 **1122회**
- 기준 전략: **평균근접(`nearestMean4`)**, **상2+하2(`twoHotTwoCold`)**
- 전략별 상위 2개 기간을 모두 반영해 각 전략 4개를 만들고, 두 전략 8개에서 **공통번호 우선**으로 최종 4개를 선정

## 2) 전략별 상위 기간(현재)

- 평균근접: **28, 108**
- 상2+하2: **332, 268**

## 3) 변경 로직 최종 4개 백테스트 결과

- 최신 기준 예시 최종 4개: **3, 5, 7, 43**
- `>=1` 적중률: **48.48%**
- 평균 적중: **0.582**
- 최대 연속 미적중: **8회**

## 4) 해석 (판단용)

- 단일 최고 윈도우 기준 수치(평균근접 48.93%, 상2+하2 49.20%)보다,
  **2개 기간 통합 최종 4개(48.48%)**는 소폭 낮지만 안정화된 합성 규칙으로 운영 가능
- 공통번호 우선 규칙은 설명 가능성이 높아 운영/설명 측면에서 유리
- 다만 연속 미적중 8회가 있어, 보수 운영 시 필터 또는 보조 룰(제외/가중)을 추가하는 것이 좋음

## 5) 현재 번호 선정 규칙 (간단)

1. 각 전략에서 상위 기간 2개를 선택
2. 기간별 점수를 합산해 전략별 4개 번호 선정
3. 두 전략 8개에서 공통번호 우선
4. 부족분은 점수 높은 번호로 채워 최종 4개 확정

## 6) 실행 명령

```bash
cd frontend
npm run backtest:accumulated-numbers
```
